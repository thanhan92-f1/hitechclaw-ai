# Support

This document explains where to ask questions, where to report bugs, and how to route licensing or security topics for HiTechClaw AI.

---

## Where to Go

### Questions and implementation help

Use **GitHub Discussions** for:

- setup and deployment questions
- usage guidance
- workflow design questions
- architecture discussion
- operator best practices
- roadmap conversation and general product feedback

### Bugs and defects

Use **GitHub Issues** when you can describe a reproducible defect, regression, or broken behavior.

Please include:

- what happened
- what should have happened
- steps to reproduce
- environment details
- severity and impact
- logs or screenshots when available

### Feature requests

Use **GitHub Issues** with the feature request template for scoped enhancements that fit the project.

### Security problems

Use the private disclosure route described in [`SECURITY.md`](SECURITY.md). Do not open public issues for exploitable security reports.

### Licensing and commercial rights

Review [`LICENSE`](LICENSE) and [`COMMERCIAL-LICENSE.md`](COMMERCIAL-LICENSE.md). If you need commercial rights, contact the repository owner / copyright holder for a separate written agreement.

---

## What Makes a Good Support Request

The fastest support requests usually include:

- exact version or commit
- installation method such as Docker Compose or manual runtime
- environment details such as OS, browser, Node.js version, and container version
- exact API request or UI path involved
- relevant logs, screenshots, or console output
- a concise statement of business or operational impact

## Maintainer Guidance

In general:

- move usage questions to Discussions
- keep Issues focused on actionable engineering work
- keep security matters private
- request reproduction data early when reports are incomplete
- keep licensing discussions off public issues when they involve commercial negotiation
