# Licensing Guide

## Overview

`HiTechClaw AI` is provided under a custom non-commercial license.
Individuals, companies, and organizations may use the repository only for
strictly non-commercial purposes unless separate written commercial permission
has been granted by the copyright holder.

The controlling files are:

- `../LICENSE`
- `../NOTICE`
- `../COMMERCIAL-LICENSE.md`

If any summary conflicts with the canonical license text, `../LICENSE` controls.

## Non-Commercial Use Only

Allowed non-commercial use includes:

- personal learning and private experimentation
- academic or educational work
- internal research, testing, and evaluation
- internal non-commercial use by a company or organization

These examples are descriptive, not exhaustive. The key rule is that the use
must not be intended to create commercial advantage, generate revenue, support
paid delivery, or enable commercialization.

## Commercial Use Requires Written Permission

A separate written commercial license is required before you:

- sell, sublicense, commercialize, or monetize the repository or derivative works
- use the repository in paid consulting, implementation, integration, deployment, training, or support
- use the repository in paid client work, contract delivery, managed services, or outsourcing
- include the repository in a paid product, hosted service, SaaS offering, bundle, subscription, or marketplace listing
- use the repository in a workflow intended to generate revenue or provide commercial advantage

Until such permission is granted in writing, all commercial rights remain reserved.

## Important Clarification

Availability of this repository on GitHub, npm, or any other public source
distribution channel does not grant any commercial right.

Public visibility does not change the licensing terms. Downloading, forking,
cloning, or referencing the repository does not create permission for paid use,
commercial redistribution, or commercial service delivery.

## Practical Examples

### Usually allowed

- a developer testing the install flow in a private lab
- a student using the repository for coursework or research
- an internal engineering team evaluating the deployment model without selling it
- a company reviewing the repository for architecture learning or internal assessment

### Not allowed without permission

- deploying the repository for a paying customer
- charging for integration, customization, onboarding, or support based on this repository
- bundling the repository into a commercial SaaS or managed platform
- reselling the repository, modified versions, or derivative automation packages
- using the repository as part of a revenue-generating consulting engagement
